# AI Compare

An AI-powered product comparison SaaS that helps users make smarter decisions. Compare any products in seconds, get structured results, and an AI-driven recommendation telling you what to choose and why.

**Live Demo:** [ai-compare-saas.vercel.app](https://ai-compare-saas.vercel.app)


## Features

- 🔍 **Instant AI Comparison** – Type any two products and get a side-by-side comparison table, generated by DeepSeek AI.
- 🏆 **Winner Recommendation** – AI picks the best option and explains why, giving users a clear decision.
- 💳 **Monetization with Stripe** – Free tier (5 comparisons/day) and Pro subscription ($9.9/month) with unlimited comparisons.
- 📊 **Structured Results** – Clean, card-based UI with feature lists, pricing, and summary.
- 📜 **History Dashboard** – View past comparisons with quick winner and summary previews.
- 🔐 **Production-Grade Security** – JWT authentication, Row Level Security (RLS), rate limiting, idempotent webhooks, Content Security Policy (CSP).
- 🚀 **Full-Stack SaaS** – Complete with landing page, login/registration, dashboard, and payment flows.

## Architecture & Security Highlights

- **Authentication** – Supabase Auth with email/password. JWT tokens are verified on every API call; user ID is extracted server-side, not trusted from the client.
- **Row Level Security** – Each user can only access their own comparisons. `authenticated` role has only read access to the `users` table; modifications require `service_role` key (server-side only).
- **Idempotent Webhooks** – Stripe checkout sessions are recorded in `users.stripe_session_id`; duplicate events are safely ignored.
- **API Rate Limiting** – Next.js middleware limits API calls per IP (especially `/api/compare` and `/api/limit`).
- **CSP Headers** – Configured in `next.config.ts` to restrict scripts and frames, mitigating XSS.
- **TypeScript Strict** – No `any` types in core logic; explicit interfaces for AI responses and database models.

## Database Schema (Supabase)

### `users` table
- `id` (uuid, PK) – links to `auth.users`
- `is_pro` (boolean) – Pro subscription status
- `stripe_session_id` (text) – for idempotent webhook processing
- `created_at` (timestamptz)

### `comparisons` table
- `id` (uuid, PK)
- `user_id` (uuid, FK → users.id)
- `items` (text) – user query (e.g., "iPhone 15 vs Samsung S24")
- `criteria` (text) – always "AI"
- `result` (jsonb) – AI response (structured comparison with winner/summary or raw text)
- `created_at` (timestamptz)

RLS policies ensure users can only read/insert their own comparisons.

## Getting Started

### Prerequisites

- Node.js >= 18
- Supabase project (with `users` and `comparisons` tables created, RLS enabled, and triggers for auto-creating `users` row on signup)
- Stripe account (test mode for development)
- DeepSeek API key (or any OpenAI-compatible API)

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/gloria238/ai-compare-saas.git
   cd ai-compare-saas
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Set up environment variables**
   Create a `.env.local` file in the root:
   ```env
   NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
   SUPABASE_SERVICE_KEY=your-service-role-key   # (used server-side for webhooks)
   DEEPSEEK_API_KEY=sk-xxxx
   STRIPE_SECRET_KEY=sk_test_xxxx
   NEXT_PUBLIC_APP_URL=http://localhost:3000
   STRIPE_WEBHOOK_SECRET=whsec_xxxx   # (from Stripe CLI or dashboard)
   ```

4. **Run database migrations**
   Execute the SQL scripts from `/supabase/migrations` (if provided) or manually create the tables and RLS policies as described in the schema.

5. **Start the development server**
   ```bash
   npm run dev
   ```
   Open [http://localhost:3000](http://localhost:3000).

6. **Stripe webhook (local testing)**
   ```bash
   stripe listen --forward-to localhost:3000/api/stripe/webhook --events checkout.session.completed
   ```
   Copy the signing secret into `STRIPE_WEBHOOK_SECRET`.

### Deployment

The app is ready to deploy on Vercel. Connect your GitHub repository, add the same environment variables, and set the framework to **Next.js**. The `NEXT_PUBLIC_APP_URL` should be your production URL (e.g., `https://ai-compare-saas.vercel.app`).

After deployment:
- Update Supabase **Site URL** and **Redirect URLs** to match your production domain.
- Set up a production Stripe webhook endpoint pointing to `https://your-domain.com/api/stripe/webhook`.
